# Spec / Onboarding

Onboarding is a guided setup conversation, separate from ordinary chat but powered by the same thread engine.

Flow:

1. identity and station
2. key relationships
3. commitments and goals
4. current state
5. areas for help

Rules:

- ask one bounded question at a time
- prefer concrete facts over vague self-description
- probe once when an answer is sparse, then move on
- go deep on people and commitments
- create files at section transitions, not after every answer
- record only what the user actually provided

Completion is explicit. Do not infer completion merely because `profile/index.md` exists.
